# Step1. 他者と世界の認識を分かち合う

> [!NOTE]
> **本章の位置づけ：**
>
> 本 Step では、Step0 で扱った個人の探索を、チームとしての探索へと接続する。<br>
> Project Sprint において、個人が行動し出力できるだけでは、まだプロジェクトは成立しない。<br>
> 各メンバーがそれぞれ異なる方向へ進んでいる状態では、「プロジェクトが前進している」とは言えないからである。<br>
> ここでは、他者との相互作用を通じて共通了解を形成し、「何に向かって進んでいるのか」をチームとして共有しながら探索を進める状態を扱う。
>
> **制作側の目的：**
>
> - Project Sprint において、チームとして探索と収束を成立させるためには、他者との相互作用と共通了解が不可欠であることを示す。
> - 人は互いを完全には理解できないという制約を前提としたうえで、それでも共に前進するための関係性をどのように形成するのかを明らかにする。
> - 個人の自由な探索だけではなく、他者からの応答を受け取りながら、自らの行動や理解を調整していくことの重要性を示す。
>   
> **読者に提供する価値：**
>   
> - Project Sprint において、他者とどのように関わり、どのように共通了解を形成していくのかを理解する。
> - 異なる視点や価値観を持つメンバーと協働しながら、チームとして探索を前進させるための視点を得る。
> - 自分一人では到達できなかった理解や発展が、他者との相互作用を通じて生まれる感覚を理解できるようになる。
> - 他者と完全には分かり合えないという現実を受け入れながら、それでも共に進むための態度を理解する。

## 1. Principles：Project Sprint が理想とする状態

> [!Note]
> **本章の位置づけ：**
>
> 本章では、Project Sprint において判断や行動の基準となる価値観を示す。<br>
> 構造を理解するだけでなく、その中で何を優先すべきかを判断するための基準を提示する。
>
> **製作側の目的：**
>
> - Project Sprint が重視する価値観を宣言し、判断や行動の基準となる思想を明確にする。
>
> **読者にとっての価値：**
>
> - Project Sprint がどのような価値観にもとづいて設計されているのかを理解し、プロジェクトの中で何を重視すべきかの指針を得る。

Project Sprint では、プロジェクトを推進する際の判断基準として次の価値観を重視する。

- **計画よりも出力を**<br>
  プロジェクトは、計画ではなく出力によって前進する。<br>
  出力は、チームが世界を理解し次の行動を決めるための一次情報である。

- **正しさよりも納得を**<br>
  唯一の正解を探すことよりも、メンバーが納得して行動できることを重視する。<br>
  納得は、各メンバーが自らの行動を引き受けるための基盤となる。

- **管理よりも自律を**<br>
  チームは、一元的な管理ではなく個別的で自律的な判断によって動く。<br>
  疎結合な状態で独立して行動する各メンバーは、共通了解によって協働する。

- **巧拙よりも本気度を**<br>
  プロジェクトにおいて重要なのは、最初から上手くできることではなく、自ら関与し、行動しつづけることである。<br>
  本気の行動と出力は、チームとプロジェクトの理解を更新する。

これらの価値観は、いずれか一方のみを選択するものではない。<br>
わたしたちはそれぞれの重要性を認めたうえで、右に示したことがらにより高い価値を見出す。<br>
これらの原則は、Project Sprint の構造を通じて具体的な行動として現れる。

## 2. Assumptions

> [!Note]
> **本章の位置づけ：**
>
> 本章では、Project Sprint を理解するための前提となる、ものごとの見方を提示する。
> Project Sprint は、特定の手順を守ることで機能する仕組みではない。その背後にある考え方を理解することで、はじめて本来の効果を発揮する。
> 本章の目的は、Project Sprint を説明することではなく、Project Sprint の前提に存在する、プロジェクトを見るための一つのレンズを提示することである。
> そのレンズを通してプロジェクトを見たとき、問題や混乱の見え方は変化する。そしてその結果として、Project Sprint の構造がなぜそのようになっているのかが、説明ではなく理解として立ち上がるであろう。
>
> **制作側の目的：**
> - Project Sprint が前提としている人間観・世界観・プロジェクト観を明示し、フレームワークの思想的な基盤を共有する。
> - なぜ Project Sprint というフレームワークが成立するのか、その必然性を理解できるようにし、読者がその価値を自ら判断できる状態をつくる。
> - プロジェクトや他者との関わり方に対する態度を調整するための基準を提示する。
>
> **読者に提供する価値：**
> - Project Sprint の背景にある考え方を理解することで、フレームワークを表面的な手順としてではなく、その意図とともに捉えることができる。
> - これらの前提に照らして自分自身のプロジェクトへの向き合い方を調整することで、Project Sprint をより効果的に活用できるようになる。

### Section 1.　Project Sprint の世界観

- **世界が常に変化していることを受け入れる**<br>
  わたしたちが向き合う世界は常に変化しており、将来を完全に予測することはできない。<br>
  しかし、この事実をどのように扱うかは、プロジェクトチームの意志と選択に委ねられている。<br>
  変化をプロジェクトの前提とせず、あらかじめ定めた計画を維持してプロジェクトを進めることもできる。<br>
  しかし、変化を前提としない限り、行動によって得られる新たな情報や理解は十分に活用されない。<br>
  Project Sprint におけるプロジェクトとは、不確実な状況の中で行動とその結果を通じて更新され前進してゆく活動である。

- **人は世界のすべてを理解することはできない**<br>
  個人が持つ認識は常に限られており、世界のすべてを理解することはできない。<br>
  ひとつの視点から見えるものには限界がある。<br>
  複数の視点が多様に展開されることで、はじめて見えてくるものがある。<br>
  そのため、多様な視点が存在すること自体が、理解を深めるための前提となる。

- **理解は行動を通じて更新される**<br>
  人は考えることだけでは世界を十分に理解できない。<br>
  実際に行動し、その結果として得られる経験やアウトプットを通じて、理解は更新されていく。<br>
  行動は単なる実行ではなく、世界を知るための手段である。

- **意味は事後的にしか解釈できない**<br>
  人は自分の行動の意味や価値をあらかじめ完全に理解することはできない。<br>
  行動の結果が現れ、それを他者との対話の中で振り返ることで、はじめて意味づけがなされる。<br>
  意味は行動の前に存在するものではなく、行動の後に立ち上がるものである。

- **人間の認識は完全には共有できない**<br>
  人はそれぞれ異なる経験や知識を持っており、世界の理解は常に個人的で部分的なものである。<br>
  そのため、認識を完全に一致させることはできない。<br>
  重要なのは、認識を一致させることではなく、違いを前提にしながら共に行動できる状態をつくることである。<br>
  Project Sprint では、この状態を「共通了解」と呼ぶ。

- **人は納得したときに行動を引き受ける**<br>
  意思決定は論理的な正しさだけでは成立しない。<br>
  人は、自らの判断に納得したときにはじめて、その行動を主体的に引き受けることができる。<br>
  ここでいう納得とは、その主体が何らかの根拠をもって「よい」と感じることができている状態を指す。<br>
  納得に基づく意思決定は、個人の自律的な行動を生み出す。


### Section 2.　Project Sprint における基本概念

> [!NOTE]
> **本章の位置づけ：**
>
> 本章では、Project Sprint を構成する要素とその関係性に焦点を当てる。<br>
> Project Sprint は個別の手順やルールの集合ではなく、複数の要素が相互に作用することで機能する構造である。<br>
> そのため、各要素を個別に理解するだけではなく、それらがどのように結びつき、探索と収束のリズムを生み出しているのかを捉えることが重要である。<br>
> 本章は、その関係性を明らかにし、Project Sprint 全体を構造として理解するためのものである。<br>
> 各要素の詳細な運用については Practical Guide に委ね、本章では構造そのものの理解に重点を置く。
>
> **制作側の目的：**
> - Project Sprint を構成する要素とその関係を示し、プロジェクトが探索と収束のリズムを通じて前進する構造を明らかにする。
>
> **読者に提供する価値：**
> - Project Sprint が、出力・対話・振り返りといった要素を通じて、探索と収束のリズムをどのように生み出しているのかを理解できる。
> - プロジェクトが不確実な状況の中でも前進し続ける仕組みを、手順ではなく関係性として把握できるようになる。

- **プロジェクトとは何か**<br>
  プロジェクトとは、不確実な状況の中で価値を探索し実現するための活動である。<br>
  価値や正解はあらかじめ確定しているものではなく、行動を通じて見出されていく。<br>
  そのため、プロジェクトは計画を実行するものではなく、理解を更新しながら前進していくものとして捉えられる。<br>
  また、プロジェクトそのものも、価値を実現するために設けられた一つの収束の単位である。

- **価値・成果・出力の関係**<br>
  Project Sprint では、プロジェクトを次の三層の構造として捉える。
  - 出力：プロジェクトメンバーが行動によって作成物を生み出したり情報を得たりすること
  - 成果：出力によって世界に現れた結果
  - 価値：成果が社会や顧客にもたらす意味<br>

  以下それぞれについて詳述する。<br>
  - **出力とは何か**<br>
    出力とは、プロジェクトメンバーが行動を通じて作成物や情報を生み出す行為である。<br>
    Project Sprint において、プロジェクトの前進は出力から始まる。各メンバーが行動し、その出力の結果である具体的な作成物をチームに持ち込むことで、個人の活動がプロジェクトに接続される。言語による対話だけでは認識が揃いにくい場合でも、具体的に目に見える作成物が共有されることで、問題意識や課題が可視化される。これにより、個人の中にあった気づきや違和感がチームに持ち込まれ、共通了解の形成が促される。<br>
    また、出力は単に成果物を生み出すための手段であるだけはない。行動の過程で得られるひらめきや違和感といった情報そのものが重要であり、それらが次の行動を生み出す契機となって探索を継続させる。<br>
    出力は事後的な意味づけの対象でもある。個人の行動は、それ単体ではプロジェクトにおける位置づけや価値が明確ではない。出力の後に行われる対話を通じて、はじめてその意味が解釈され、プロジェクトの中での役割が認識される。このように、出力は理解を更新し、探索を継続し、収束へとつなげる起点となる行為である。
  - **成果とは何か**<br>
	  成果とは、出力という行為の結果として、プロジェクト内外に現れた変化である。<br>
    作成物の生成にとどまらず、状況や認識、関係性などが変化した状態を含む。<br>
    成果は観測可能な事実として存在し、プロジェクトの進行に伴って積み重なっていく。
  - **価値とは何か**<br>
    価値とは、成果に対して外部のステークホルダーが見出す意味や便益である。プロジェクトの成功とは、この価値を顧客に届けることであると言える。<br>
    価値は成果そのものに内在するものではなく、それを受け取る主体によって解釈される。同じ成果であっても、その意味づけは状況や立場によって異なりうる。プロジェクトチームにとって、価値は事前に完全に定義されるものではなく、行動と出力を通じて徐々に明らかになっていく仮説である。

- **探索と収束とは何か**<br>
プロジェクトの本質は探索にある。<br>
不確実な状況においては、価値や正解は一度の判断や計画によって確定するものではなく、行動を通じて見出されていく。プロジェクトは、行動によって得られた結果をもとに理解をいったん収束させ、次の行動を選択することで前進する。Project Sprint は、この探索と収束の連続によってプロジェクトが進むものと捉える。<br>
その前提として、多様な個人が個々の能力を発揮してそれぞれの視点から世界を解釈し、その結果をプロジェクトに持ち込むことが不可欠である。こうした多様性が存在してはじめて、収束という行為が意味を持ち、プロジェクトはチームにとってよりよいと感じられる方向へと進んでいく。

- **定例会議とは何か**<br>
定例会議とは、収束を一定の間隔で繰り返すことでプロジェクトに前進のリズムを意図的に与えるための装置である。<br>
そのため、定例会議は以下の3つの条件を満たすように設計されなくてはならない。<br>
  - 短期的：プロジェクト全体のスケジュール感から見て相対的に短期であること
  - 定期的：人間の既存の習慣に沿った一定の期間が置かれること（日次、週次、隔週等）
  - 反復的：一定の型が存在すること（進行方法、アジェンダの種類や形式等）<br>

  定例会議においては、各メンバーの出力をもとに対話が行われることで、個人の中にあった理解がチームのものとして再構成される。この過程を通じて、チームとしての共通了解が更新され、次の行動の方向が形づくられる。<br>
  また、このような収束を一定のリズムで繰り返すことにより、各メンバーが自律的に行動できる環境と習慣が形成される。

- **チームとは何か**<br>
プロジェクトチームとは、異なる視点や経験を持ちながら、共通了解によってゆるやかにつながる個人の集合である。<br>
各メンバーはそれぞれ独立した判断主体として自律的に行動し、その出力がチームの理解を更新しつづける。<br>
チームとは、認識を統一する場ではなく、多様な認識を持ち寄りながらプロジェクトを前進させるための関係性である。

- **共通了解とは何か**<br>
共通了解とは、メンバーが完全に同じ理解を共有している状態ではない。それぞれが異なる理解を持ちながらも、互いの考え方や状況をおおよそ把握し、チームとして共通の目的に向かって統合的な行動を取れる状態を指す。<br>
認識を一致させるのではなく、違いを前提としながら行動を共にするための基盤であり、個人の多様性を発揮するための土台とも言える。<br>
共通了解は固定されたものではなく、出力と対話を通じて継続的に更新される。

- **納得とは何か**<br>
納得とは、その主体が何らかの根拠をもって「よい」と感じることができている状態を指す。<br>
複雑で不確実な状況では、唯一の正解を事前に特定することは難しい。そのため、意思決定において重要なのは正しさではなく、各メンバーがその判断に納得し、自らの行動として引き受けられることである。このとき、共通了解によって形成された状況認識や方向性が、納得の前提となる。納得に基づく意思決定は、個人の自律的な行動を生み出す。<br>
共通了解はチームとして行動するための前提であり、納得は個人がその行動を引き受けるための条件である。

- **プロジェクトをどう認識するか**<br>
Project Sprint においては、便宜上プロジェクトを以下の3つの側面に分けて認識することがある。<br>
  - プログレス：成果の実現に関する要素にフォーカスする。
  - プロセス：プロジェクトの環境や進め方に関する要素にフォーカスする。
  - チーミング：チームの関係性や自律性に関する要素にフォーカスする。

  プロジェクトにおける個々の活動を、これらの3つの側面それぞれから解釈することで、個々の活動がプロジェクトの中で持つ意義や効果をより正確かつ多面的に認識できるようになる。この認識をプロジェクトチームで共有することにより、各メンバーは自身の行動を状況に応じて調整し、プロジェクト全体としての前進に寄与することができる。

## Mindset : 他者と探索を共有する主体としての態度

> [!NOTE]
> **本章の位置づけ：**<br>
> 本章では、Step0 で扱った「個人として探索へ参加する態度」を、他者との協働へと接続する。<br>
> ここでは、他者と完全には分かり合えないという前提に立ちながら、それでも共通了解を形成し、共に探索を進めるための態度を扱う。
>
> **制作側の目的：**
> - Project Sprint において、チームとして探索と収束を成立させるためには、他者の出力を受け取り、それに応答しながら、自らの行動を調整していく姿勢が不可欠であることを示す。
> - 探索とは単独で完結するものではなく、相互作用を通じて予想外の発展や価値が立ち上がる営みであることを共有する。
>
> **読者に提供する価値：**
> - Project Sprint において、他者の出力や視点とどのように向き合い、どのように協働へ接続していくのかを理解する。
> - 単に個人として行動するだけでなく、チームとして探索を深めるための視点を得る。
> - 他者との相互作用を通じて、自分一人では到達できなかった理解や発展が生まれる感覚を理解できるようになる。

Project Sprint において、チームとは、認識が完全に一致した集団ではない。各メンバーは、それぞれ異なる経験や知識、関心を持っており、世界の見え方も異なっている。そのため、他者を完全に理解することはできない。しかし、Project Sprint では、この違いを排除しない。むしろ、多様な視点が存在することそのものが、探索を豊かにすると考える。そのため、重要なのは、他者を完全に理解することではなく、他者の出力に応答しながら、共に探索を続けることである。

Project Sprint では、この協働のあり方を理解するために、ジャズやインプロ（即興演劇）の実践を参照する。これらの活動では、全体の進行が最初から完全に決まっているわけではない。各参加者が自ら表現を行い、それに対して他の参加者が応答することで、全体が形づくられていく。

このとき重要なのは、他者の表現を単に評価したり修正したりすることではない。まず相手の表現を受け取り、その上で自らの表現を重ねることである。インプロではこれを “Yes, and” の原則と呼ぶ。

Project Sprint においても同様に、メンバーは他者の出力を観察し、それに応答することで、プロジェクトの理解と方向性を更新していく。この過程では、予想外の発展や、自分一人では到達できなかった理解が生まれる。他者との相互作用によって、探索は個人の限界を超えて広がっていく。

同時に、他者と関わることは、必ずしも快適なことではない。意図が伝わらなかったり、自分とは異なる価値観や判断に直面したりすることもある。それでも、対話と応答を繰り返しながら共通了解を形成し、探索を継続することが、Project Sprint におけるチームの前進を支えている。

探索とは、自分だけで世界を更新することではない。他者との相互作用を通じて、チームとして世界の見え方を更新していくことである。
