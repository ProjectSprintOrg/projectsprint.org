# Step0. 個人が世界を更新する

> [!NOTE]
> **本章の位置づけ：**
>
> 本Stepでは、Project Sprint におけるもっとも基本的な単位として、「個人」に着目する。<br>
> ここでは、プロジェクト全体の構造把握やチームとしての成果実現よりも前に、各メンバーが自ら行動し、出力し、世界に働きかけられる状態を実現することを目指す。<br>
> Project Sprint の出発点は、正しい成果を出すことではなく、まず個人が探索に参加できることである。
>
> **制作側の目的：**
> - Project Sprint において重要となる、行動・出力・観察・提案といった個人の基本的な態度を共有する。
> - 個人が世界に働きかけ、その結果を観察することで、自らの理解を更新しつづける存在であることを示す。
> - 成果への適合やチームとしての統合よりも前に、「まず行動できること」「出力できること」を重視する。
>
> **読者に提供する価値：**
> - Project Sprint において、個人としてどのように探索へ参加すればよいかを理解する。
> - 成果や役割に過度に縛られることなく、自ら行動し、出力し、プロジェクトを前進させるための視点を得る。
> - 自らの行動によって世界の見え方が更新される感覚を得ることで、Project Sprint の探索的なプロジェクト観を体験的に理解できるようになる。

## 1. Assumptions

> [!NOTE]
> **本章の位置づけ：**
>
> 本章では、Project Sprint に参加する個人が、まずどのような前提に立つ必要があるかを示す。<br>
> ここでは、チームとして成果を出すことよりも前に、各メンバーが自ら行動し、出力し、世界に働きかけられる状態を重視する。
>
> **制作側の目的：**
>
> - Project Sprint の出発点として、個人が主体的に行動し、自らの観察や違和感をプロジェクトに持ち込める状態を成立させる。
> - 「正しい答えを出すこと」よりも、「行動し、出力し、世界を更新すること」がプロジェクトの前進につながるという前提を共有する。
>
> **読者に提供する価値：**
>
> - Project Sprint において、まず個人としてどのように行動すればよいかを理解する。
> - 成果や役割に過度に縛られることなく、自ら行動し、出力し、探索へ参加するための視点を得る。

わたしたちは、世界を完全に理解することはできない。<br>
人はそれぞれ、自らの経験や知識、関心にもとづいて世界を認識している。<br>
そのため、理解は考えることだけでは更新されない。<br>
実際に行動し、その結果を観察することで、はじめて世界の見え方は変化する。

Project Sprint において重要なのは、最初から正しい成果を出すことではない。<br>
まず必要なのは、自ら行動し、出力し、その結果を観察できるようにすることである。<br>

出力は、必ずしも完成された成果物である必要はない。<br>
違和感、途中経過、試行錯誤、失敗、ひらめきもまた、プロジェクトを前進させるための重要な情報である。

各メンバーは、自らの観察や思考をチームへ持ち込み、プロジェクトの理解を更新する主体として扱われる。<br>
 「正しいかどうか」をチームに問うのではなく、「何を観察し、何を試し、何を出力したか」「その上で、何が最善であると考えるか」をチームに提案することが重要となる。

Project Sprint における探索とは、世界に働きかけ、その結果によって自らの理解を更新し続けることである。

## 2. Structure：探索と収束の基本サイクル

> [!NOTE]
> **本章の位置づけ：**
>
> 本章では、Project Sprint の最小単位となる探索と収束のサイクルを示す。<br>
> Workshopで体験したプロセスの構造をコンパクトに捉え直すことを目的とする。
>
> **制作側の目的：**
>
> - 複雑な全体像に入る前に、Project Sprint の核となる最小の動きを明確にする。
> - 読者がWorkshopで体験した内容に対して意味づけを行い、「何が起きていたのか」を個別の行動や手順ではなく、全体の構造として理解できる状態をつくる。
>
> **読者に提供する価値：**
>
> - Workshopで体験したプロセスを、感覚的な理解から構造的な理解へと引き上げることができる。
> - Project Sprint の基本的な動き（出力・対話・収束・次の行動）を自分の言葉で説明できるようになる。

Project Sprint は、探索と収束の小さなサイクルを繰り返すことでプロジェクトを前進させるフレームワークである。このフレームワークは、複数の要素が連動することで、プロジェクトを前進させる構造になっている。

各メンバーは、それぞれの視点や関心にもとづいて行動し、生み出された作成物や気づきを出力する。出力されたものはチームに持ち込まれ、対話を通じて意味づけが行われる。この過程で、個人の中にあった理解はチームのものとして再構成される。その結果として共通了解が更新され、チームとしての次の方向性が形づくられる。更新された共通了解をもとに、各メンバーは納得に基づいて次の行動を選択する。そして新たな探索が始まり、再び出力が生み出される。

Project Sprint では、この一連の流れを定例会議を単位として繰り返す。定例会議は、探索によって広がった理解を一時的に収束させ、次の行動を可能にするための場である。

このように、Project Sprint におけるプロジェクトは、出力と対話を通じて探索と収束を反復することで前進する。Workshop で体験したプロセスは、この基本的なサイクルを体験するためのものである。

### Section 1.  出力と対話による探索と収束

プロジェクトの本質は探索にある。<br>
不確実な状況においては、価値や正解は一度の判断や計画によって確定するものではなく、行動を通じて見出されていく。

プロジェクトは、行動によって得られた結果をもとに理解をいったん収束させ、次の行動を選択することで前進していく。この過程は、プロジェクトチームの行為によって世界の見え方を更新しつづける営みでもある。<br>
Project Sprint は、この探索と収束の連続によってプロジェクトが進むものと捉える。その前提として、多様な個人がそれぞれの視点から世界を解釈し、その結果をプロジェクトに持ち込むことが不可欠である。

出力は、Project Sprint の構造における起点となる探索行為である。<br>
各メンバーが行動することによって生み出される作成物や気づきは、プロジェクトに新たな情報をもたらす。この情報がなければ、チームは状況を更新することができない。<br>
出力は、単に成果物を生み出すためのものではなく、世界を理解するための手段である。行動の過程で得られるひらめきや違和感は、プロジェクトにおける重要な情報として扱われる。<br>
Project Sprint においては、この出力が継続的に行われることによって探索が維持される。

対話は、出力によってもたらされた情報に意味を与える収束プロセスである。
各メンバーの出力は、そのままでは個人の中に閉じたものである。それがチームに持ち込まれ、対話を通じて解釈されることで、はじめてチームの理解として共有される。<br>
この過程において、認識の一致は前提とされない。重要なのは、それぞれの理解を持ったまま、状況をおおよそ把握し、行動の方向を揃えられる状態をつくることである。<br>
対話は、個人の出力を共通了解へと接続する役割を持つ。

出力と対話の繰り返しは、探索と収束のリズムであり、同時に分化と統合のリズムでもある。<br>
Project Sprint は、このリズムを維持しつづけることで、プロジェクトの理解と行動を更新しつづける構造である。

### Section 2.  定例会議という装置

定例会議は、収束を一定の間隔で強制的に起こすことで、プロジェクトに前進のリズムを意図的に与える装置である。<br>
プロジェクトは変化しつづけるため、収束は自然には発生しない。<br>
しかし、チームとして統合的に前進するためには、一時的な収束を定期的に発生させる必要がある。<br>
収束によって、成果物は形になりやすくなり、次の行動が起こりやすくなる。

各メンバーの出力は定例会議に持ち込まれ、対話を通じて意味づけが行われる。その結果、共通了解が更新され、次の行動の方向が形づくられる。<br>
これは単なる情報共有ではなく、プロジェクトメンバーの意志によって収束のあり方を定める行為である。<br>
この収束を定期的に繰り返すことで、探索と収束の反復が維持される。

### Section 3.  収穫祭としての定例会議

Project Sprint では、定例会議を「収穫祭」として捉える。

各メンバーの試行錯誤や挑戦の結果は、単なる報告ではなく、プロジェクトの理解を更新するための成果として扱われる。<br>
それらは対話を通じてチームに回収され、プロジェクトの中で意味づけされる。<br>
収穫は出力の結果そのものではなく、その意味づけによって成立するのである。

この過程において、個人の行動は評価される対象ではなく、次の行動を生み出すための材料として扱われる。<br>
各メンバーは、自らの出力がどのように位置づけられたかを踏まえ、納得に基づいて次の行動を選択する。

定例会議を収穫祭として再定義することにより、個人の試行錯誤は肯定的に回収され、チームの理解として統合される。<br>
各メンバーは、行動が強制的な収束を迎えた結果を、適切な名残惜しさを抱きつつ収穫として寿ぐことになる。<br>
その結果、次の行動に必要な材料が各メンバーの手元に揃い、各自が自らの意志で次の行動を選び取り引き受ける。

収穫祭とは、行動の結果を一度収束させ、それを意味として受け取り直す場である。<br>
Project Sprint における定例会議は、この収穫を通じて納得と自発性を生み出す構造である。

### Section 4.  振り返り

Project Sprint における振り返りとは、これまでの出力と気づきに対してチームで意味づけを行い、プロジェクトの認識と構造を更新するプロセスである。

振り返りの主眼は、過去を評価することではなく、過去の経験を手がかりにチームが未来へ向き直ることにある。<br>
振り返りは、認識を一致させる場ではなく、次の行動を可能にする共通了解を再形成する場である。

振り返りは、次の二つの側面を同時に扱う。

- **環境認識の更新**

  プロジェクトが世界をどのように理解しているかを更新する。<br>
  これまでの出力によって得られた成果や気づきをもとに、「わたしたちは何を目指しているのか」と「そのために何が必要か」を再解釈する。<br>
  この過程を通じて、価値や目標といった統合的な観点と、役割や期待値といった分化的な観点の両方が調整される。

- **プロセスの更新**

  プロジェクトの進め方そのものを更新する。<br>
  運営や仕組みに関する問題意識やアイデアをチームで共有し、それらに対して意味づけを行い、納得したうえでプロジェクトに反映する。<br>
  これにより、プロジェクトのプロセスは状況に応じて継続的に最適化される。

振り返りを通じて、プロジェクトは単に前進するだけでなく、その進み方そのものを更新しつづける。<br>
それは、統合と分化の関係を調整しながら、プロジェクトの構造を再構成しつづける営みである。

## 3. Mindset : 探索に参加する主体としての態度

> [!NOTE]
> **本章の位置づけ：**
>
> 本章では、Project Sprint の探索と収束のサイクルにおいて、各メンバーがどのような態度で参加することが求められるのかを示す。<br>
> ここで扱うのは、成果や役割よりも前に、世界に働きかけ、自ら行動を引き受ける主体としての基本的な姿勢である。
>
> **制作側の目的：**
>
> - Project Sprint において重要となる、誠実さ、責任、自律性、インテグリティといった個人の基本的な態度を共有する。
> - 単にタスクを処理するのではなく、自ら世界に関与し、その結果を引き受けながら探索へ参加することの重要性を示す。
> - Workshopで体験した対話や出力の背景にある協働の考え方を理解するための補助線とする。
>
> **読者に提供する価値：**
>
> - Project Sprint において、個人がどのような態度で行動し、出力し、他者の出力に応答し、プロジェクトへ関与することが求められるのかを理解する。
> - 単なる作業者ではなく、主体的に探索へ参加する存在として、自らの行動を捉えられるようになる。

Project Sprint において、各メンバーは単に与えられた役割を遂行する存在ではない。それぞれが、自ら世界を観察し、違和感を持ち、行動し、その結果をプロジェクトへ持ち込む主体として扱われる。そのため、Project Sprint では、まず個人が自らの行動を引き受けることが重視される。

ここで重要なのは、最初から正しい答えを出すことではない。不完全であっても、自ら観察し、考え、行動し、その結果を誠実に出力することが大切である。

誠実さとは、正しく振る舞うことではなく、自らが観察した事実や感じた違和感を隠さずプロジェクトへ持ち込むことである。また、Project Sprint における責任とは、失敗しないことではない。自らの行動と出力を、自分自身の意思によって引き受けることである。

この意味で、Project Sprint は、メンバーに受動的な遂行者であることを求めない。それぞれが探索へ参加する主体として、自ら世界に関与することを求める。

探索とは、世界を変えることであると同時に、自らが変わり続けることでもある。
